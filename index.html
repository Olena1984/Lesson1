• git config --global user.name - налаштування, щоб задати своє ім'я перед початком використання git, нообхідне для того щоб знати виконавця
  • git config --global user.email - налаштування, за допомогою якого записуємо електронну пошту виконавця
  • git init - команда, що дозволяє створити нове сховище Git. Завджи запускається при створенні нового проекту
  • git status - дозволяє переглянути поточний стан сховища
  • git add - команда за допомогою якої додаються зміни у робочий файл
  • git commit - команда, що фіксує зміни коду у сховищі
  • git log - команда, що допомагає переглянути записану історію проекту від останніх змін до попередніх у зворотному порядку
  • git checkout - команда, що дозволяє перейти до існуючої гілки, або перейти до нової
  • git branch - інструмент керування гілками. Може виводити список існуючих гілок, створювати нові гілки, вилучати чи перейменовувати їх
  • git push -команда, що використовується для того, щоб зв'язатися з іншим сховищем, обчислити що є у нашій локальній базі даних, а у віддаленній нема, а потім надіслати різницю до іншого сховища
  • git pull - команда за допомогою якої Git отримує зміни із заданого віддаленого сховища, а потім пробує злити їх до поточної гілки
  • git merge - допомагає об'єднати одну чи більше гілок до поточної гілки
  • git clone - команда, що створює локальну робочу копію вихідного коду з віддаленого сховища.

.gitignore файл - файл, що повідомляє git не показувати певні папки і файли приховані від перегляду.

  • git revert - використовується для запису деяких нових комітів, щоб скасувати дію деяких попередніх комітів (часто лише помилкових).
  • git reset - це команда, яку ми використовуємо, коли хочемо перемістити репозиторій назад до попереднього коміту, відкидаючи будь-які зміни, зроблені після цього коміту.
  • git rebase -Перебазування один із двох інструментів Git для впровадження змін з однієї гілки до іншої. Такі самі можливості надає команда git merge (злиття). Операція злиття фіксує зміни, завжди рухаючись уперед по історії проекту, тоді як перебазування дозволяє ефективно її переписувати.
  • git cherry-pick - бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміта в поточній гілці. Ця можливість корисна в ситуації, коли потрібно забрати парочку коммітів з іншої гілки, а не зливати гілку цілком з усіма змінами, що внесли до неї.
  • git stash - команда, що дозволяє на час здати в архів чи відмінити зміни зроблені в робочій копії, щоб потім можна було застосувати їх. Функція корисна коли ми відклали зміни і ще не готові для створення коміту
